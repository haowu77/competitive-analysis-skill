# i18n Behavior

## Supported output languages
- `en`, `zh`, `ja`, `ko`, `es`, `fr`, `de`

## Language selection
1. If `--lang` is explicit, always use it.
2. If `--lang auto`, detect language from selected source:
- `--lang-source brief`: detect from `--brief`
- `--lang-source input`: detect from `--input-json` text
- `--lang-source both`: combine both (default)
3. If detection is ambiguous or unavailable, fallback to `en`.

## What is localized
- Worksheet names
- Column headers
- Default Summary 3-row template
- Enum values (Threat, Category, Confidence, status-like fields)
- Validation warning messages printed to console

## What is not automatically translated
- Free-text competitor descriptions already provided by the user/input JSON
- URLs, raw citations, and numeric values

## Mixed-language handling
- The script uses character/token distribution to detect dominant language.
- If language signal is too close across candidates, fallback to `en` for safety.
